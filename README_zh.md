# 數獨求解器 (numpy)

本專案是一個使用 `numpy` 的簡單數獨求解與檢查程式。包含求解器、基本 I/O 工具，以及多次運行記錄輸出功能。英文說明位於 `README.md`，中文說明在本檔案。

## 功能

- 從 `sudoku.txt` 載入數獨題目（若檔案不存在則隨機生成）。
- 使用回溯法檢查並嘗試解題。
- 結果會儲存在 `sudoku_results.txt`。

## 需求

- Python 3.8+
- 參考 `requirements.txt` 安裝相依套件。

## 使用方法

1. 準備 `sudoku.txt`（9x9，空格分隔，0 表示空格），若無則會隨機生成。
2. 執行：

```powershell
python sudoku_solver.py
```

遵循互動提示選擇執行次數與是否從上次結果繼續。

## 程式運作說明

本專案包含數獨讀取、檢查與回溯求解功能。主要模組與功能：

- `load_sudoku(file_path)`：從 `sudoku.txt` 讀取 9x9 格式，若檔案不存在則隨機生成。
- `generate_sudoku()`：隨機產生 9x9，值為 0–9（0 表示空格）。注意：隨機生成題目未必為有效且可解的數獨。
- `is_valid(grid, row, col, num)`：檢查在 `(row, col)` 放 `num` 是否違反行、列或 3x3 區塊規則。
- `solve_sudoku(grid, errors)`：使用回溯法（遞迴）填入空格，`errors` 為 9x9 陣列，標記在搜尋過程中沒有候選數的格子。
- `is_solvable(grid)`：複製原題並執行求解，回傳 `(solvable, filled_grid, errors)`。
- `save_to_file(...)`：將每次執行的結果以可讀文字行追加至 `sudoku_results.txt`。
- 主程式 `main(...)`：支援多次執行、列印題目與解答或錯誤位置，並可將結果寫入檔案。

## 實作細節

- 求解器使用基本回溯（深度優先搜尋）。會嘗試 1–9 的數字並在衝突時回溯。此方法對一般題目可用，但未做效能最佳化或唯一性檢查。
- `errors` 陣列是用來輔助顯示在某條搜尋路徑中無候選數的格子，不能作為整體不可解的嚴格證明。
- `sudoku_results.txt` 目前為純文字格式，如需機器可讀，建議改為輸出 JSON。

## 注意事項

- 輸入格式：`sudoku.txt` 必須為 9 行，每行 9 個整數（0 表示空格），以空白分隔。格式錯誤會被檢查並提示。
- 隨機題目：`generate_sudoku()` 產生的題目多半為無效或存在衝突，僅適合快速測試。
- 效能：回溯法最差情況為指數時間。若要大量執行或處理困難題，建議加入啟發式演算法或使用更高效的演算法（如 dancing links）。
- 唯一性：本求解器找到一個解即可，並不保證該解是唯一解。
- 顏色與清螢幕：程式使用 ANSI 顏色碼與 Windows `cls` 指令；部分終端可能無法顯示顏色，必要時可移除色碼或改用跨平台函式庫。
- 編碼：讀檔時使用 UTF-8，若您使用不同編碼請更新 `open` 的參數。

## 想要改進？

- 若您想改進求解器，可考慮：
  - 在回溯前加入限制傳播（消去候選數）。
  - 使用啟發式：選擇候選數最少的格子（MRV）、候選數排序，或改用精準覆蓋（dancing links）。
  - 將 `sudoku_results.txt` 改為 JSON/CSV 以利分析。
